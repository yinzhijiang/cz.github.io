github+hexo博客的搭建过程

By [object Object]

Published 2018-03-05

  1. 首先在GitHub账户上建立自己的项目仓库（⚠️点击setting下拉至GitHub Pages ，将source改为master branch 保存之后上面就会出现一串的地址，点击就可以看到相应的web页面信息，这里为什么要改为master branch呢，是因为GitHub pages要以一个分支做基准，master是主分支）

  2. 在终端查看是否安装了git & npm 命令 : git –version & npm –version

  3. 新建文件夹blog， cd blog进入文件夹全局安装hexo 命令 npm install hexo -g ，检查是否安装成功 hexo-v

  4. 初始化blog文件夹。hexo init

  5. 安装组件npm install

  6. hexo g

  7. 运行 hexo s 此时端口号是4000如果页面没有出来的话有可能是端口号被占用了，修改端口号的命令：hexo server -p 端口号

  8. 将Hexo与Github page联系起来，设置Git的user name和email（如果是第一次的话）

    1. cd 到 blog文件夹下，命令行 vi _config.yml，下滑至最后，修改deploy
    2. 执行生成静态页面。hexo generate or hexo g
    3. 执行配置页面。 hexo deploy or hexo d

    如报错 找不到git

    执行 npm install hexo-deployer-git –save

    再次执行

    hexo g

    hexo d

  9. 更改主题的话去hexo选取自己喜欢的主题根据链接进入GitHub，有详细的步骤解说。
